## Design Context

### Users
UCP is used by Unity developers, technical artists, tooling engineers, CI operators, and AI coding agents working against real Unity projects. They use the docs and website in a task-oriented context: to discover capabilities quickly, understand safe workflows, and move from setup to automation without UI clutter or ambiguity.

### Brand Personality
Refined, technical, reliable. The interface should feel calm, deliberate, and trustworthy rather than flashy or trend-driven, with enough polish to communicate product maturity for serious automation work.

### Aesthetic Direction
Refined technical. Favor restrained contrast, subtle violet emphasis, crisp structure, and low-noise surfaces over loud gradients or decorative chrome. Dark and light themes should both feel precise and subdued, with navigation chrome blending into the page instead of calling attention to itself.

### Design Principles
1. Prioritize workflow clarity over command sprawl or decorative complexity.
2. Keep technical surfaces dense but calm, with strong hierarchy and bounded output.
3. Use polish to reinforce trust: subtle motion, restrained accents, and clean alignment.
4. Prefer maintainable shared structure over duplicated UI configuration.
5. Avoid generic AI-looking patterns; every visible detail should feel intentional and utility-driven.
